# 跟踪

跟踪窗口提供了一个用于查看和导出数据的界面。 用户可以通过工具栏上的按钮保存数据，以便进一步分析或存档。

![alt text](../../../media/um/trace/image.png)

> [!INFO]
> 目前，跟踪的最大存储容量为 50,000 条记录。 超过此限制时，最旧的数据将被自动删除。

## 覆盖模式

使用下方按钮在覆盖模式和滚动模式之间切换。
![alt text](../../../media/um/trace/image-5.png)

在覆盖模式下，当超过最大存储容量时，跟踪窗口将覆盖最旧的数据。
![ow](../../../media/um/trace/ow.gif)

## 拖放BLF/ASC文件

您可以直接将BLF或ASC跟踪文件拖放到跟踪窗口中进行离线分析。 文件将被解析并分页显示。

![拖放](../../../media/um/trace/drag-drop.png)

### 通道映射

首次加载文件时，将出现一个通道映射对话框。 这允许您将日志文件的通道号映射到项目配置的CAN设备。 如果设备附加了DBC数据库，消息名称和信号将自动解码。

![通道映射](../../../media/um/trace/channel-map.png)

### 分页

大型跟踪文件会分页显示。 您可以配置页面大小（每页10K~1M帧），并使用工具栏控件在页面之间导航。

### 帧详情面板

单击任意行可在跟踪窗口底部展开详情面板（Wireshark风格）：

- **左侧**：帧字段信息（ID、方向、类型、时间戳、十六进制转储）
- **右侧**：带有解码值的信号表（信号名称、物理值、原始值、位位置、枚举标签）

左右面板之间的分隔线可以拖动以调整大小。

![详情面板](../../../media/um/trace/detail-panel.png)

## 筛选

### 按设备筛选

跟踪窗口支持按设备、信号名称和信号值进行筛选。

> [!NOTE]
> 选择所有设备或不选择任何设备具有相同的效果。

![alt text](../../../media/um/trace/image-3.png)

## 按消息类型筛选

![alt text](../../../media/um/trace/image-4.png)

- CAN - 接收 CAN 相关数据
- LIN - 接收 LIN 相关数据
- UDS - 接收 UDS 相关数据
- ETH - 接收以太网相关数据

## 支持的导出格式

- Excel - 以 Microsoft Excel 格式导出数据
- ASC (ASCII) - 以 ASCII 格式导出数据，兼容各种 CAN 分析工具
- [功能请求](./../../dev/feature.md)

## 列信息

追踪窗口包含以下列（可通过列设置面板进行配置）：

- **#**：顺序索引编号
- **时间**：自录制/回放开始以来经过的时间（秒）
- **Δt(s)**：连续帧之间的时间差（秒）
- **UTC 时间**：帧的绝对 UTC 时间戳
- **名称**：消息名称（如果可用，从 DBC 解码）
- **数据**：十六进制格式的原始数据字节
- **DIR (方向)**：指示方向（Tx 为发送，Rx 为接收）
- **ID**：消息标识符
- **DLC (数据长度码)**：数据长度码
- **LEN (长度)**：实际数据长度（字节）
- **类型**：消息类型（例如 CAN、CAN-FD、扩展）
- **通道**：通信通道
- **设备**：设备名称

列的可见性、顺序和宽度可通过列配置面板自定义，并在会话之间保持不变。

![column-config](../../../media/um/trace/column-config.png)

当相应的硬件通道绑定到[数据库](../database.md)时，这些列信息有助于用户快速了解和分析系统的运行状态。

> [!INFO]
> 帧内的信号值只能在跟踪窗口暂停时查看
